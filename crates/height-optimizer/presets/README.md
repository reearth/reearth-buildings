# Calibration presets

`HeightConfig` TOML files. Pass via
`height-optimizer compare --candidate <file>`.

The full background — methodology, findings, why we picked v3 — lives
in the parent crate's [../README.md](../README.md), written in paper
form (background / objectives / methods / results / future work).

| file                              | what it is |
|-----------------------------------|------------|
| `baseline.toml`                   | Verbatim dump of `HeightConfig::default()`. **Production already runs the v3 values**, so this file is effectively equivalent to `candidate-tuned-v3.toml` (it is regenerated by `dump-config`). |
| `candidate-conservative.toml`     | Mild bumps (office 30→40, hotel 25→35, m/floor 3.3). Safe but the gains are small. |
| `candidate-tall-cbd.toml`         | Aggressive (office/commercial 30→60, m/floor 3.5). Helps CBDs, regresses suburbs. |
| `candidate-tuned-v2.toml`         | Bumps the dense_urban table + raises suburban_min. **Regression** at Iiyama / Tsukuba — kept as a documented anti-pattern. |
| `candidate-tuned-v3.toml`         | The adopted one: dense_urban catch-all bucket only, plus class.office 40, plus m/floor 3.3. |

Now that `HeightConfig::default()` is v3, comparing the existing
candidates against the new baseline lets you see the diff as
"further trials beyond v3" — they remain useful starting points
for the next round of search.
